# Akiko Insights Seed (v1)
目的：KPIから「人が動ける判断」に変換するための“頭脳モデル”の種。
方針：「誰でも言える」は禁止。数字単体ではなく “組み合わせ・偏り・運用設計” まで見る。

---

## 判断軸（Akikoの視点）
- **数字の正しさ**：そもそも測れてる？定義・列・サンプルが成立してる？
- **行動可能性**：その指摘は「今日何をする」に落ちる？
- **原因の粒度**：人／案件（タグ）／プロセス／ナレッジ／システムのどこか？
- **再現性**：たまたまではなく、繰り返し起きる構造か？
- **費用対効果**：改善コストに見合う？最小手数で直せる？
- **リスク**：法務・炎上・コンプラ・顧客離反の爆弾になってない？
- **現場運用**：SV/QAが回せる設計？“やりっぱなし”にならない？

---

## 気づき（20〜30個）

### 1) AHT（短い＝正義ではない）
1. **AHTが低いのにCSATが低い** → “早く切ってる/雑/確認不足”。改善は「速さ」ではなく**確認フロー**。
2. **AHTが低いのにFCRが低い** → “一回で終わらせてない”。**再問い合わせ**が増えてコスト増。
3. **AHTが低いのにEscalationが高い** → “自分で抱えないで投げてる”。**一次完結の権限・手順不足**。
4. **AHTが低いトップ層が固定化** → “裏技/テンプレ偏重”の可能性。品質監査で**抜け項目**を見る。
5. **AHTが高いのにCSATが高い** → “丁寧は価値”。短縮より**再現性**（どう丁寧にやってるか）を型化。

### 2) CSAT（平均より分布・中央値）
6. **平均CSATだけ見ると誤認**。分布が二極化してるなら“当たり外れ”問題。**80未満帯の原因**を先に潰す。
7. **90-100が多いのに0-79も多い** → 運用が不安定。**新人比率/案件タイプ/時間帯**で割る。
8. **CSATが少数の低評価で崩れる** → “重大不満”が混ざってる。**低評価コメント分類**が最短ルート。
9. **CSATが目標未達でもFCRが高い** → 解決はしてるが“態度/説明/期待調整”が弱い。**話し方・共感・一言**を改善。
10. **CSATが高いのにエスカが高い** → “丁寧に引き継いでるだけ”。根本は**一次で処理できる知識/権限**。

### 3) FCR（unknownは設計ミス）
11. **FCR unknownが多い** → 現場の問題じゃなく**計測設計の欠陥**。まず列/定義/抽出ロジックを直すのが最優先。
12. **FCRが高いのにAHTが異常に低い** → “切断・転送・テンプレ誘導”の疑い。**通話後処理/転送率**も見る。
13. **FCRが低いのにAHTが高い** → “調べてるのに解決できない”。**ナレッジの検索性**か**権限不足**。
14. **FCRが週末だけ落ちる** → バックオフィス不在/承認待ち。**休日運用の例外手順**が必要。
15. **FCRが高いのに再入電が増える** → “FCR定義が甘い”。**同一顧客の短期再問い合わせ**を別指標で持つ。

### 4) SLA（列が無い＝現場ではなくデータ側）
16. **SLA列が無い/欠損** → まず“測れない”。改善施策より先に**出力定義の確定**が必須。
17. **SLAが悪いのにCSATが高い** → 期待値調整が上手い可能性。ただし放置すると炎上。**遅延時テンプレ**を標準化。
18. **SLAが悪化しているのにAHTが改善** → “急いでるのに捌けてない”。**需要過多（人員）**か**ルーティング**。
19. **SLAが一部の時間帯だけ悪い** → シフト/休憩設計ミス。**WFM改善が最短**。
20. **SLAとEscalationが連動** → 初動遅れ→不満→エスカ。**一次で詰まってる理由**を潰す。

### 5) Escalation（“増える理由”が必ずある）
21. **エスカ率が高いのにCSATがそこそこ** → “エスカ基準が緩い/保身”。**L1で止めるチェックリスト**が効く。
22. **特定案件タグでエスカ集中** → 個人の能力ではなく**プロダクト/ポリシー**。顧客向け説明文・FAQを整備。
23. **新人のエスカ率が高い** → 教育不足より**判断基準の文章化不足**。分岐フローチャートが最短。
24. **ベテランのエスカが増えた** → ルール変更・権限縮小・システム劣化。現場のせいにしない。
25. **エスカ率は“数”より“質”**。正しいエスカは必要。問題は“不要エスカ”なので**理由カテゴリ**で分ける。

### 6) エージェント評価（ランキングの罠）
26. **ランキングは“上位/下位”より“変化量”**。急落した人が最優先。原因は環境変化が多い。
27. **成績が安定してる人は“再現可能な型”を持つ**。その人を褒めるだけで終わらせず、**行動を抽出して標準化**。
28. **低評価の人に研修を当てる前に、案件配分の偏り**を疑う。難易度が偏ると全てが崩れる。
29. **数値が良いのに監査でNGが多い** → “成果のためにルールを削ってる”。長期的に事故るので早期是正。
30. **改善の基本は“1人を直す”より“原因を減らす”**。トップ3原因を潰すと、全員の数値が上がる。

## 使い方（次ステップ用メモ）
- このSeedを元に「insightルール」や「taskテンプレ」を増やしていく。
- 1つの気づきは必ず「今日やるタスク（所要時間・期限・担当）」に落とす。
